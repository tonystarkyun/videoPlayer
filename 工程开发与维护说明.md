# 字幕时间轴播放器工程开发与维护说明

## 工程目标

这是一个本地轻量播放器工程，用来把：

- 本地视频文件
- 本地字幕 Markdown 文件

组合成一个可交互的字幕时间轴播放器。

当前核心功能：

- 解析 `## Segments`
- 在右侧渲染时间轴字幕列表
- 点击字幕跳转视频
- 自动生成并叠加 VTT 字幕
- 支持默认项目配置和快速切换项目

## 目录结构

- [index.html](/E:/myvideo/subtitle-player/index.html)
  页面结构
- [styles.css](/E:/myvideo/subtitle-player/styles.css)
  页面布局和样式
- [app.js](/E:/myvideo/subtitle-player/app.js)
  前端主逻辑，包括加载配置、解析渲染、点击跳转、状态同步
- [parseSegments.js](/E:/myvideo/subtitle-player/parseSegments.js)
  Markdown `## Segments` 解析和 WebVTT 生成
- [project.config.json](/E:/myvideo/subtitle-player/project.config.json)
  当前默认项目配置
- [export-default-vtt.mjs](/E:/myvideo/subtitle-player/export-default-vtt.mjs)
  根据默认配置导出 `.vtt`
- [convert-md-to-vtt.mjs](/E:/myvideo/subtitle-player/convert-md-to-vtt.mjs)
  将任意 Markdown 转为 `.vtt`
- [serve.py](/E:/myvideo/subtitle-player/serve.py)
  本地静态服务，支持 MIME 和视频 Range 请求
- [start.ps1](/E:/myvideo/subtitle-player/start.ps1)
  一键启动入口，可更新配置、导出 VTT、启动服务并打开浏览器

## 核心数据格式

当前工程依赖的字幕 Markdown 必须包含：

```md
## Segments

[00:00:00.000 - 00:00:11.620] 段落-0 第一条字幕
[00:00:12.020 - 00:00:14.550] 段落-0 第二条字幕
```

解析逻辑由 [parseSegments.js](/E:/myvideo/subtitle-player/parseSegments.js) 负责，正则会提取：

- 开始时间
- 结束时间
- 字幕文本

然后生成：

- 前端渲染使用的 segments 数组
- WebVTT 字幕文件

## 页面运行链路

1. [start.ps1](/E:/myvideo/subtitle-player/start.ps1) 根据参数决定是否更新 [project.config.json](/E:/myvideo/subtitle-player/project.config.json)。
2. [start.ps1](/E:/myvideo/subtitle-player/start.ps1) 调用 [export-default-vtt.mjs](/E:/myvideo/subtitle-player/export-default-vtt.mjs) 导出默认 `.vtt`。
3. [start.ps1](/E:/myvideo/subtitle-player/start.ps1) 启动 [serve.py](/E:/myvideo/subtitle-player/serve.py)。
4. 浏览器打开 [index.html](/E:/myvideo/subtitle-player/index.html)。
5. [app.js](/E:/myvideo/subtitle-player/app.js) 读取 [project.config.json](/E:/myvideo/subtitle-player/project.config.json)。
6. [app.js](/E:/myvideo/subtitle-player/app.js) 加载视频和 Markdown。
7. [app.js](/E:/myvideo/subtitle-player/app.js) 调用 [parseSegments.js](/E:/myvideo/subtitle-player/parseSegments.js) 解析字幕。
8. 前端渲染右侧列表，并将生成的 VTT 挂到 `<track>` 上。

## 维护时最关键的技术点

### 1. 不要再用 `python -m http.server`

原因：

- `.js` / `.mjs` 在某些环境会返回错误 MIME
- MP4 seek 需要 Range 支持，而默认服务行为不稳定

统一使用：

```powershell
python E:\myvideo\subtitle-player\serve.py --port 8765
```

或者直接用：

```powershell
powershell -ExecutionPolicy Bypass -File E:\myvideo\subtitle-player\start.ps1
```

### 2. 视频 seek 依赖 Range 请求

[serve.py](/E:/myvideo/subtitle-player/serve.py) 已经支持：

- `Accept-Ranges: bytes`
- `206 Partial Content`
- `Content-Range`

如果后续你替换服务端，一定要保留这部分能力。

### 3. `project.config.json` 是默认项目入口

当前前端默认只认一个配置文件：

- [project.config.json](/E:/myvideo/subtitle-player/project.config.json)

如果以后要支持多项目切换，有两种方向：

- 继续单配置，但通过脚本动态改写
- 升级成“项目列表 + 切换 UI”

当前版本已经支持通过 [start.ps1](/E:/myvideo/subtitle-player/start.ps1) 传参后自动改写配置。

## 推荐的迭代方向

### 第一阶段：增强现有播放器

- 增加上一条/下一条更明显的视觉反馈
- 支持键盘快进/快退
- 支持定位到当前字幕时自动置中
- 支持记住上次播放位置
- 支持字幕搜索结果高亮

### 第二阶段：增强字幕数据能力

- 支持导出 `.srt`
- 支持字幕文本编辑后重新导出 `.md` / `.vtt`
- 支持按句、按段、按说话人分组
- 支持时间轴微调

### 第三阶段：增强项目管理能力

- 支持多个视频项目配置
- 支持最近打开列表
- 支持拖拽导入视频和 Markdown
- 支持扫描目录自动识别同名 `.mp4` / `.md` / `.vtt`

## 代码维护建议

### 前端逻辑

优先维护 [app.js](/E:/myvideo/subtitle-player/app.js) 的这几块边界：

- 配置加载
- 视频源加载
- Markdown 解析结果渲染
- 点击跳转
- 自动高亮同步

不要把所有状态都混在一个函数里，后续如果功能继续增加，建议把它拆成：

- `config`
- `player`
- `timeline`
- `subtitle-track`

四个模块。

### 解析逻辑

[parseSegments.js](/E:/myvideo/subtitle-player/parseSegments.js) 最适合保持纯函数化。

后续如果要支持更多 Markdown 格式，建议增加新解析器，不要直接把现有规则改成非常复杂的一个正则大杂烩。

### 服务端

[serve.py](/E:/myvideo/subtitle-player/serve.py) 现在虽然不复杂，但职责很明确：

- 正确 MIME
- 正确静态资源分发
- 正确视频 Range

如果后续你改成 Node 服务，也要对这三个目标做回归验证。

## 变更前建议检查

1. 有没有影响 `project.config.json` 的字段格式。
2. 有没有影响 `Range` 请求。
3. 有没有影响 `## Segments` 解析。
4. 有没有影响点击字幕时的 seek 逻辑。
5. 有没有引入浏览器缓存导致“改了但页面没生效”的问题。

## 回归测试建议

每次改完后至少检查：

1. 页面能否正常打开。
2. 默认视频和 Markdown 能否自动加载。
3. 右侧是否能显示片段数。
4. 点击第 1 条、第 10 条、第 50 条字幕，视频是否跳转到正确时间。
5. 浏览器中字幕轨是否正常显示。
6. 右侧长列表滚动时，布局是否稳定。

## 常用命令

导出默认项目 VTT：

```powershell
node E:\myvideo\subtitle-player\export-default-vtt.mjs
```

将任意 Markdown 转成 VTT：

```powershell
node E:\myvideo\subtitle-player\convert-md-to-vtt.mjs "E:\path\demo.md"
```

一键启动默认项目：

```powershell
powershell -ExecutionPolicy Bypass -File E:\myvideo\subtitle-player\start.ps1
```

一键切换并启动新项目：

```powershell
powershell -ExecutionPolicy Bypass -File E:\myvideo\subtitle-player\start.ps1 `
  -Title "示例项目" `
  -VideoPath "E:\path\demo.mp4" `
  -MarkdownPath "E:\path\demo.md"
```
