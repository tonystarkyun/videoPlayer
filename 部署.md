# 字幕时间轴播放器部署指南

## 结论

如果你要在另一台设备上使用这个工程，**基本可以直接把整个 `subtitle-player` 工程目录拷贝过去**，但前提是目标机器具备运行环境。

这个工程**不依赖 `npm install`，也不依赖 `pip install`**。当前依赖全部来自：

- Node.js 运行本地 `.mjs/.js` 脚本
- Python 标准库运行本地静态服务
- PowerShell 运行启动脚本
- 浏览器加载页面并选择本地文件夹

也就是说，它是“**可直接拷贝，但不是完全零依赖**”。

## 部署方式

### 方式 1：推荐

把整个目录直接复制到另一台机器，例如：

```text
E:\tools\subtitle-player
```

然后在新机器上直接运行：

```powershell
powershell -ExecutionPolicy Bypass -File E:\tools\subtitle-player\start.ps1
```

### 方式 2：只复制核心文件

如果你不想带上说明文档和缓存目录，也可以只复制这些核心文件：

- `app.js`
- `convert-md-to-vtt.mjs`
- `export-default-vtt.mjs`
- `index.html`
- `package.json`
- `parseSegments.js`
- `project.config.json`
- `serve.py`
- `start.ps1`
- `styles.css`

## 不需要复制的内容

这些不是运行必需：

- `.vs`
- `__pycache__`
- 旧测试文件
- 你自己的临时导出文件

## 部署依赖清单

### 必需依赖

#### 1. PowerShell

用途：

- 运行 [start.ps1](/E:/myvideo/subtitle-player/start.ps1)

建议版本：

- Windows PowerShell 5.1 或 PowerShell 7+

最低建议：

- 只要支持当前脚本中的 `Get-NetTCPConnection`、`Start-Process`、`ConvertFrom-Json` 即可

#### 2. Node.js

用途：

- 运行 [export-default-vtt.mjs](/E:/myvideo/subtitle-player/export-default-vtt.mjs)
- 运行 [convert-md-to-vtt.mjs](/E:/myvideo/subtitle-player/convert-md-to-vtt.mjs)

建议版本：

- Node.js 18+

原因：

- 工程使用 ES Module
- `.mjs` / `import` 语法需要稳定的 ESM 支持

#### 3. Python

用途：

- 运行 [serve.py](/E:/myvideo/subtitle-player/serve.py)

建议版本：

- Python 3.10+

原因：

- 当前服务脚本依赖 Python 3 的标准库
- 不需要安装额外第三方包

#### 4. 浏览器

用途：

- 打开播放器页面
- 使用“选择本地文件夹”
- 播放本地视频

建议浏览器：

- Microsoft Edge
- Google Chrome

原因：

- 页面使用了 `webkitdirectory` 选择文件夹
- Chromium 系浏览器对这类本地目录选择支持更稳定

## 不需要安装的依赖

下面这些当前都不需要：

- `npm install`
- `node_modules`
- `pip install`
- 第三方 Python 框架
- 第三方前端构建工具

## 目标设备系统建议

### 最推荐

- Windows 10 / Windows 11

原因：

- 当前启动入口是 [start.ps1](/E:/myvideo/subtitle-player/start.ps1)
- 当前日常使用流程默认围绕 PowerShell

### 如果是 macOS / Linux

前端页面和 Python 服务本身是可迁移的，但当前这个一键启动脚本不是直接通用的。

你需要改为手动执行：

```bash
node /path/to/subtitle-player/export-default-vtt.mjs
python3 /path/to/subtitle-player/serve.py --port 8765
```

然后手动打开：

[http://127.0.0.1:8765/subtitle-player/index.html](http://127.0.0.1:8765/subtitle-player/index.html)

如果你明确要跨 macOS / Linux 长期用，建议后面再补一个 `.sh` 启动脚本。

## 部署前检查清单

把工程复制到新机器前，检查：

1. 目标机器是否安装了 Node.js
2. 目标机器是否安装了 Python 3
3. 目标机器是否可用 PowerShell
4. 目标机器是否有 Edge 或 Chrome
5. 你是否保留了 `subtitle-player` 目录中的核心脚本

## 部署后首次验证清单

在新机器上按这个顺序验证。

### 第一步：检查命令是否存在

```powershell
node -v
python --version
$PSVersionTable.PSVersion
```

如果三条都能正常输出版本，说明基础环境齐了。

### 第二步：启动工程

假设工程目录是：

```text
E:\tools\subtitle-player
```

运行：

```powershell
powershell -ExecutionPolicy Bypass -File E:\tools\subtitle-player\start.ps1
```

### 第三步：检查浏览器页面

打开：

[http://127.0.0.1:8765/subtitle-player/index.html](http://127.0.0.1:8765/subtitle-player/index.html)

确认页面能正常显示。

### 第四步：测试目录工作流

准备一个测试目录，例如：

```text
E:\demo-media
  demo-01.mp4
  demo-01.md
  demo-02.mp4
  demo-02.md
```

然后在页面里：

1. 点击“选择本地文件夹”
2. 选中 `E:\demo-media`
3. 检查左侧下方是否出现视频列表
4. 点击某个视频项
5. 检查右侧字幕是否加载
6. 点击右侧字幕，检查视频是否跳转

### 第五步：测试字幕开关

确认：

- 默认视频内字幕不显示
- 打开“显示视频字幕”后才显示字幕

## 另一台设备是否必须复制视频和字幕文件

### 如果只是复制工程

只复制工程本身，并不会把你原机器上的视频和字幕带过去。

所以你还需要在新机器上准备：

- 视频文件
- 对应字幕 Markdown 文件

### 最推荐的做法

工程和素材分开管理：

- 工程目录单独放
- 视频和字幕放在你自己的媒体目录

例如：

```text
E:\tools\subtitle-player
E:\media\course-a
E:\media\course-b
```

这样你只要把工程复制过去，再把自己的媒体目录也准备好，就能直接在页面里选目录使用。

## 目录工作流下的素材要求

每个目录中建议这样组织：

```text
某个目录
  demo-01.mp4
  demo-01.md
  demo-02.mp4
  demo-02.md
```

要求：

- 视频和 Markdown 同名
- Markdown 中有 `## Segments`

当前页面会自动扫描同名配对：

- 有 `.mp4` 但没有同名 `.md` 的文件会被忽略
- 有 `.md` 但没有同名视频的文件也会被忽略

## 默认配置在新机器上的作用

[project.config.json](/E:/myvideo/subtitle-player/project.config.json) 只用于：

- 默认项目启动
- `start.ps1 -Title/-VideoPath/-MarkdownPath/-VttPath` 写入默认项目

但你现在的主要工作流已经改成：

- 启动工程
- 在页面中选本地文件夹

所以即使默认配置没改，只要页面能打开，你仍然可以直接选新机器上的本地目录使用。

## 常见部署问题

### 1. 页面打开了，但目录选择后没有列表

优先检查：

- 文件是否真的是同名 `.mp4 + .md`
- `.md` 中是否有 `## Segments`
- 浏览器是否是 Edge / Chrome

### 2. 页面能开，但点击字幕视频不跳转

优先检查：

- 是否使用了工程自带 [serve.py](/E:/myvideo/subtitle-player/serve.py)
- 是否错误用了 `python -m http.server`

不要用：

```powershell
python -m http.server 8765
```

要用：

```powershell
powershell -ExecutionPolicy Bypass -File E:\tools\subtitle-player\start.ps1
```

或者：

```powershell
python E:\tools\subtitle-player\serve.py --port 8765
```

### 3. 提示端口被占用

可以换端口：

```powershell
powershell -ExecutionPolicy Bypass -File E:\tools\subtitle-player\start.ps1 -Port 8766
```

### 4. PowerShell 提示脚本执行策略问题

用这个命令运行：

```powershell
powershell -ExecutionPolicy Bypass -File E:\tools\subtitle-player\start.ps1
```

## 最终部署建议

如果你只是把它迁移到另一台 Windows 机器，推荐按这个清单做：

1. 安装 Node.js 18+
2. 安装 Python 3.10+
3. 确认有 PowerShell
4. 确认有 Edge 或 Chrome
5. 复制整个 `subtitle-player` 目录
6. 准备你的本地视频和同名字幕 Markdown
7. 运行 [start.ps1](/E:/myvideo/subtitle-player/start.ps1)
8. 在页面中选择素材目录开始使用

## 一句话结论

**可以直接拷贝工程过去，但目标机器至少要有 Node.js、Python、PowerShell 和一个 Chromium 浏览器；不需要 npm 安装，也不需要 pip 安装。**
