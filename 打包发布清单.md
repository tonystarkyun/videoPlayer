# 打包发布清单

## 目标

这份清单用于把当前工程拷贝到另一台机器时，尽量少带无关文件，同时保证启动、目录扫描、字幕跳转、VTT 生成都能正常工作。

## 推荐打包方式

最省事的方式不是挑文件，而是**直接复制整个 `subtitle-player` 目录**。

如果你想手动精简，只带下面“必须文件”即可。

## 可以不带的内容

这些通常不需要复制：

- `.vs`
- `__pycache__`
- 临时测试输出文件
- 你本机的示例视频、示例字幕、临时导出 VTT

## 推荐发布目录结构

推荐在新机器上保持这样的目录结构：

```text
E:\tools\subtitle-player
```

素材目录单独放，例如：

```text
E:\media\project-a
E:\media\project-b
```

这样工程和素材不会混在一起，后续升级工程时也更方便。

## 新机器必须具备的环境

### 必需

- Node.js 18+
- Python 3.10+
- PowerShell
- Edge 或 Chrome

### 不需要

- `npm install`
- `pip install`
- `node_modules`

## 推荐发布动作

每次准备拷贝到另一台机器前，按这个顺序做：

1. 确认 [start.ps1](/E:/myvideo/subtitle-player/start.ps1) 能正常启动工程。
2. 确认 [serve.py](/E:/myvideo/subtitle-player/serve.py) 仍然支持视频跳转。
3. 确认 [convert-md-to-vtt.mjs](/E:/myvideo/subtitle-player/convert-md-to-vtt.mjs) 还能把 Markdown 转成 VTT。
4. 清理不需要的临时文件。
5. 复制整个 `subtitle-player` 目录，或者只复制“必须文件”。

## 新机器首次验证清单

拷过去之后，优先做这几步：

1. 打开 PowerShell，检查环境：

```powershell
node -v
python --version
$PSVersionTable.PSVersion
```

2. 双击 [一键启动.bat](/E:/myvideo/subtitle-player/一键启动.bat)，或者运行：

```powershell
powershell -ExecutionPolicy Bypass -File E:\tools\subtitle-player\start.ps1
```

3. 页面打开后，选择一个包含同名 `.mp4 + .md` 的目录。
4. 检查左侧视频下方是否出现视频列表。
5. 点击某一项，检查右侧字幕是否加载。
6. 点击某条字幕，检查视频是否跳转。

## 一键启动.bat 的用途

[一键启动.bat](/E:/myvideo/subtitle-player/一键启动.bat) 是为了让你在新机器上更省事：

- 可以直接双击启动
- 也可以在命令行里传参数

例如指定端口：

```cmd
E:\tools\subtitle-player\一键启动.bat -Port 8766
```

## 最终建议

如果你只是自己在多台 Windows 设备之间迁移，最简单的策略就是：

1. 安装 Node.js、Python、PowerShell、Chrome/Edge
2. 直接复制整个 `subtitle-player` 目录
3. 双击 [一键启动.bat](/E:/myvideo/subtitle-player/一键启动.bat)
4. 在页面里选择素材目录

这套方式最稳，也最省维护成本。
