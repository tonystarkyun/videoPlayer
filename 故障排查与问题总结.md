# 字幕时间轴播放器故障排查与问题总结

## 背景

本工程的目标是读取本地视频和对应的字幕 Markdown 文件，从 `## Segments` 中解析时间轴与字幕内容，在页面中实现：

- 左侧视频播放
- 右侧时间轴字幕列表
- 点击字幕后跳转到对应时间播放
- 自动生成并叠加 VTT 字幕

在开发和联调过程中，先后遇到了几类问题。

## 问题 1：页面一直停在“正在读取默认配置”

### 现象

- 页面能打开，但视频、字幕源、片段数一直是未加载。
- 右侧字幕列表为空。
- 浏览器地址已经是本地地址，例如 `http://127.0.0.1:8765/subtitle-player/index.html`。

### 根因

最开始使用了：

```powershell
python -m http.server 8765
```

但在当前 Windows 环境里，这种方式把 `.js` / `.mjs` 当成了 `text/plain` 返回给浏览器。浏览器不会执行模块脚本，于是前端逻辑根本没有启动。

### 解决方案

- 新增工程自带服务脚本 [serve.py](/E:/myvideo/subtitle-player/serve.py)
- 显式声明 `.js` / `.mjs` / `.json` / `.vtt` 等文件的 MIME 类型
- 不再使用 `python -m http.server`

### 经验

本地静态页面里只要用了 `type="module"`，就不要假设系统默认 HTTP 服务一定会返回正确的 MIME 类型。

## 问题 2：点击字幕时，页面高亮变化了，但视频总是从头开始播放

### 现象

- 点击右侧任意字幕，高亮会移动到被点击项。
- 但左侧视频不会跳转到该时间点，而是仍然从 `00:00` 附近播放。

### 根因

视频跳转依赖浏览器对 MP4 的范围请求能力。浏览器在 seek 时通常会重新发起 `Range` 请求，要求只取视频的一部分。

旧版服务不支持 `Range`，返回的是：

- `200 OK`

而不是：

- `206 Partial Content`

因此浏览器无法正确按字节范围跳转视频，只能从头开始读。

### 解决方案

在 [serve.py](/E:/myvideo/subtitle-player/serve.py) 中增加了对 `Range` 请求的处理：

- 解析 `Range: bytes=...`
- 返回 `206 Partial Content`
- 返回 `Content-Range`
- 返回 `Accept-Ranges: bytes`
- 仅输出指定字节范围

### 经验

本地播放器只要要支持 MP4 seek，就必须验证服务端是否支持 Range。不要只看“视频能播放”，还要看“视频能否正确跳转”。

## 问题 3：改完服务后，页面还是继续走旧逻辑

### 现象

- 明明已经修改了服务脚本，但浏览器行为和旧版本一样。
- 命令行里能看到端口已经被旧 Python 进程占用。

### 根因

旧的 `8765` 端口服务没有停掉，浏览器请求仍然打到了旧进程，而不是新版服务。

### 解决方案

- 使用 `Get-NetTCPConnection` 查端口占用
- 定位旧进程 PID
- 用 `Stop-Process -Id <PID>` 停掉旧服务
- 再重新启动新版服务

### 经验

调试本地 Web 工程时，端口冲突是高频问题。看到行为不符合预期时，要先确认请求是不是打到了正确的服务进程。

## 问题 4：字幕列表和视频布局互相影响

### 现象

- 右侧字幕列表很长时，页面整体滚动体验不理想。
- 左右两栏的可视区域边界不清晰。

### 根因

布局初版对左右两栏的滚动职责划分不够清楚。

### 解决方案

调整 [styles.css](/E:/myvideo/subtitle-player/styles.css)：

- 桌面端将左右两栏都限定在视口内
- 右侧字幕列表独立滚动
- 左侧视频区域保持稳定显示

### 经验

播放器类页面不要让“整页滚动”和“字幕列表滚动”混在一起，否则交互体验会很乱。

## 问题 5：PowerShell 输出中文乱码

### 现象

- 某些脚本或配置内容在 PowerShell 终端里显示为乱码
- 浏览器和文件本身通常是正常的

### 根因

这是终端编码显示问题，不一定是文件内容损坏。

### 解决方案

- 脚本里的控制台输出尽量使用 ASCII / 英文
- 文档和配置文件继续使用 UTF-8

### 经验

要区分“终端显示乱码”和“文件编码真的坏了”。前者常见于 Windows 终端，后者才需要修文件。

## 当前稳定结论

当前这版工程已经形成了稳定闭环：

- Markdown `## Segments` 可解析
- 可生成 `.vtt`
- 页面能正确加载视频和字幕
- 点击字幕可驱动视频跳转
- 本地服务支持模块脚本和视频 Range 请求

## 之后遇到问题时优先排查的顺序

1. 先确认浏览器访问的是不是新版服务端口。
2. 再确认 `serve.py` 是否仍由当前工程启动。
3. 再确认 `.md` 文件中是否存在 `## Segments`。
4. 再确认视频和 Markdown 路径是否正确写入 [project.config.json](/E:/myvideo/subtitle-player/project.config.json)。
5. 再确认 MP4 请求是否支持 `206 Partial Content`。
